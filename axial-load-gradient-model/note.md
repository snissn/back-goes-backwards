* strengthening in isolation fails,

**Formal description**

> The psoas behaves as a _distributed torsion cable_, with segmental phase offsets rather than uniform shortening.

Again, nothing swaps—but **the load path migrates laterally and inferiorly**, forcing the muscle to carry rotation internally.

# Body-Wide Load-Gradient Model (BLGM)

## 0. Purpose and scope

This model describes **how mechanical load is routed through the body** under gravity and movement, and how health vs degeneration corresponds to the **direction and monotonicity of load gradients** along the axial skeleton and its major tension networks.

It is not posture theory, not muscle-by-muscle anatomy, and not energy mysticism.  
It is a **state-dependent load-routing model**.

---

## 1. Core objects and definitions

### 1.1 Axial coordinate

Define a longitudinal coordinate $z$ along the body axis:

* $z = 0$: ground contact (feet)
* $z = 1$: skull / head mass center

All gradients are evaluated **with respect to increasing $z$** (upward).

---

### 1.2 Load function

Let:

$$L(z) = \text{net mechanical load transmitted through axial structures at height } z$$

This is not EMG, not “effort,” but **force transmission** (tension + compression + shear).

---

### 1.3 Load gradient

Define the axial load gradient:

$$G(z) = \frac{dL}{dz}$$

Interpretation:

* $G(z) > 0$: load **accumulates upward**
* $G(z) < 0$: load **dissipates upward**

This single sign determines system health.

---

## 2. The two global states (formal)

### 2.1 Healthy default state (State B / downward-biased)

**Definition**

$$G(z) < 0 \quad \text{for most of } z \in [0,1]$$

That is:

* Load is **highest low**
* Load is progressively **shed as height increases**

**Mechanical meaning**

* Tension networks carry load downward
* Spine is suspended, not compressed
* Upper segments are spared

---

### 2.2 Degenerative default state (State A / upward-biased)

**Definition**

$$G(z) > 0 \quad \text{for most of } z \in [0,1]$$

That is:

* Load **accumulates upward**
* Upper spine, neck, and skull become load sinks

**Mechanical meaning**

* Compression dominates
* Rotation leaks upward
* Degeneration risk increases

---

Lower lumbar carries load → upper lumbar spared.

* Healthy: load flows **into pelvis and thoracolumbar fascia**

Formally:

* Healthy: $G_{lats}(z) < 0$
* Pathological: $G_{lats}(z) > 0$

* Lower trapezius (LT)

Healthy:

Healthy:

* SCM stabilizes orientation only
* Head load routed downward through axial suspension

Pathological:

* SCM becomes load-bearing
* Neck becomes structural pillar

Formally:

* SCM load-bearing ⇒ positive local gradient near $z \approx 1$

* Pelvis rotates over femur without spinal involvement

**Fail (A-sign)**

* Fail both = 2

* Fail = 1

# 1) Mathematical refinement: Piecewise gradients by segment

## 1.1 Define the axial chain as discrete nodes

Let the spine be an ordered set of nodes (vertebral levels) from bottom to top:

$$v_0 = L5,\ v_1 = L4,\ \dots,\ v_n = T12,\ \dots,\ v_m = C7,\ \dots,\ v_{N} = C0$$

You can also define segment boundaries:

* **Lumbar segment** $\mathcal{L} = \{L5, L4, L3, L2, L1, T12\}$
* **Thoracic segment** $\mathcal{T} = \{T12, T11, \dots, T1, C7\}$
* **Cervical segment** $\mathcal{C} = \{C7, C6, \dots, C1, C0\}$

(You can choose slightly different endpoints; the model does not depend on the exact boundaries.)

---

## 1.2 Define an “effective load” at each node

You cannot measure spinal load directly without instrumentation, but you can define it formally as a mechanical object.

At each vertebral level $v_k$, define the **wrench** (force + moment):

$$\mathbf{w}_k =  
\begin{bmatrix}  
\mathbf{F}_k\\  
\mathbf{M}_k  
\end{bmatrix}$$

For a scalar model, define an **effective load magnitude**:

$$L_k = |F_{z,k}| + \lambda \|\mathbf{M}_k\|$$

* $F_{z,k}$: axial compressive component (or net axial load metric)
* $\|\mathbf{M}_k\|$: magnitude of bending/rotation moments at that level
* $\lambda$: weighting term (conceptual scaling)

This is the “load-through-the-system” quantity your intuition is tracking.

---

## 1.3 Define local gradients and segment gradients

Define a discrete axial gradient between adjacent levels:

$$g_k = L_{k+1} - L_k$$

Interpretation (with “upward” being increasing k):

* $g_k > 0$: load **accumulates upward** (bad)
* $g_k < 0$: load **dissipates upward** (good)

Now define a **segment-average gradient**:

$$G_{\mathcal{S}} = \frac{1}{|\mathcal{S}|-1}\sum_{k \in \mathcal{S}} g_k  
\quad \text{for } \mathcal{S} \in \{\mathcal{L},\mathcal{T},\mathcal{C}\}$$

So you get:

* $G_{\mathcal{L}}$: lumbar gradient
* $G_{\mathcal{T}}$: thoracic gradient
* $G_{\mathcal{C}}$: cervical gradient

---

## 1.4 Define “global state” as a 3-bit sign vector

Define the **state vector**:

$$\mathbf{S} = (\operatorname{sign}G_{\mathcal{L}},\ \operatorname{sign}G_{\mathcal{T}},\ \operatorname{sign}G_{\mathcal{C}})$$

Where each component is:

* $-$: downward-shedding (good)
* $+$: upward-accumulating (bad)
* (optionally $0$: mixed/neutral if near zero)

This yields 8 canonical global patterns (2³), e.g.:

* $(-,-,-)$: globally organized (your “State B default”)
* $(+,+,+)$: globally collapsed (your “State A default”)
* $(-,+,+)$: pelvis OK, thorax/neck collapsed, etc.

---

## 1.5 Define inversion loci (clinical “fault lines”)

An **inversion locus** is where the sign of the local gradient flips:

$$g_{k-1}\cdot g_k < 0$$

This gives you a mathematically clean definition of:

* “hinge points”
* “stress concentrators”
* “where symptoms localize”

These inversions often appear at junctions:

* **L5/S1**
* **T12/L1**
* **C7/T1**
* **C0/C1-2 region** (cranio-cervical junction)

Your psoas observation becomes a special case of this general idea.

# 2) Clinical mapping: Pain syndromes as gradient-inversion loci

## 2.1 Principle (non-diagnostic but predictive)

**Symptoms tend to localize at:**

1. **positive-gradient segments** (where load accumulates), and/or
2. **inversion loci** (where the system abruptly changes load-routing strategy)

This is the mechanical equivalent of stress concentration in materials.

## 2.2 Practical mapping table (hypothesis framework, not diagnosis)

| Likely locus | Typical “mechanical story” | Common complaint cluster (examples) |
| --- | --- | --- |
| **L5/S1 inversion** | Load not accepted by hips/legs → spine becomes pillar | low back “pinch,” SI irritation sensations, hip flexor grip, hamstring/adductor guarding |

* aggressive leg raises, hard psoas-focused hip flexion work

* sprint starts / jumps (not “bad,” but inherently upward-biased)

### (C) Conditional / form-dependent

These can be B-bias or A-bias depending on execution:

* squats, deadlifts, rows, push-ups, pull-ups, carries, running

Your system should train these **only after** you can reliably produce $(-,-,-)$ for short windows.

* If expansion is only anterior, thoracic segment is likely locked.

**Intervention set (Cervical/Thoracic B-bias):**

* Supine 90/90 breathing (quiet neck) 4–6 breaths
* Gentle axial elongation + exhale (no chin jam) 4 reps
* Then re-stand and do 1 backward step/side to integrate

# 1) Mathematical refinement: piecewise axial load gradients

## 1.1 Axial coordinate and “load”

Let $z$ be an axial coordinate from **ground → head**, normalized to $[0,1]$.

Define $L(z)$ as the **net mechanical load being transmitted at height $z$** through the axial system (spine + ribcage + scapulothoracic + cervical complex). “Load” here includes the net effect of tension/compression/shear transmission—not EMG.

Define the axial load gradient:

$$G(z) = \frac{dL}{dz}$$

Interpretation:

* $G(z) < 0$: load **dissipates as you go up** (downward‑shedding / tension‑routed)
* $G(z) > 0$: load **accumulates as you go up** (upward‑accumulating / compression‑routed)

This is the invariant you were pointing to.

---

## 1.2 Piecewise gradients by segment

Partition the axis into three segments with boundaries:

* **Lumbar segment**: $z \in [z_S, z_{TL}]$ (sacrum → thoracolumbar junction)
* **Thoracic segment**: $z \in [z_{TL}, z_{CT}]$ (TLJ → cervicothoracic junction)
* **Cervical segment**: $z \in [z_{CT}, 1]$ (CTJ → head)

Define piecewise constant (or average) gradients:

$$g_L = \left\langle \frac{dL}{dz} \right\rangle_{\text{lumbar}},\quad  
g_T = \left\langle \frac{dL}{dz} \right\rangle_{\text{thoracic}},\quad  
g_C = \left\langle \frac{dL}{dz} \right\rangle_{\text{cervical}}$$

**Global “good default”** corresponds to:

$$g_L < 0,\quad g_T < 0,\quad g_C < 0$$

**Global “bad default”** corresponds to:

$$g_L > 0,\quad g_T > 0,\quad g_C > 0$$

Most real humans are mixed: one segment flips before others.

---

## 1.3 Inversion loci

Define an **inversion locus** $z^\*$ where the system transitions:

$$G(z) \text{ changes sign at } z^\*$$

Examples:

* **Cervical inversion**: $g_L<0,\ g_T<0,\ g_C>0$
* **Thoracic + cervical inversion**: $g_L<0,\ g_T>0,\ g_C>0$
* **Lumbar inversion**: $g_L>0,\ g_T<0,\ g_C<0$ (less common, but exists)

In your earlier psoas example, the “good” psoas state (State B) is consistent with **unloading upper lumbar/thoracolumbar attachments** vs State A tending to **load upper lumbar / TLJ more**. (Anatomically the psoas spans multiple lumbar levels and relates to the diaphragm region; this is one reason TLJ is a common junction for state changes.) ## B) Build an algorithm that returns the smallest drill set to flip the inverted segments

### 1) Formal problem statement (set cover with signed constraints)

Let segments be $S=\{L,T,C\}$.

Let your current inversion set be:

$$P \subseteq S \quad \text{(segments currently “positive,” i.e., accumulating upward)}$$

Each drill $d$ has a signed vector:

$$\mathbf{v}_d = (v_{d,L}, v_{d,T}, v_{d,C}) \in \{-1,0,+1\}^3$$

Choose a subset $X$ of drills that makes each targeted segment move negative:

Hard constraint version:

* For each $i \in P$: $\sum_{d\in X} v_{d,i} \le -1$
* For each $j \notin P$: $\sum_{d\in X} v_{d,j} \le 0$ (don’t worsen a segment that’s currently OK)

Objective:

$$\min |X|$$

Tie‑breakers: minimize risk / complexity.

Because there are only **3 segments**, the exact minimal solution is trivial by enumeration.

---

### 2) Exact minimal selection by enumeration (recommended)

This is the cleanest method: check all subsets of size 1, then 2, then 3.

**Pseudocode (exact, tier‑gated):**

```python
def vec_add(a,b):
    return {s: a[s] + b[s] for s in SEGMENTS}

    # cost = number of drills + small penalties for higher tier / conditional risk
    return sum(d["cost"] for d in subset) + 0.1*max(d["tier"] for d in subset) + 0.2*sum(d["risk"] for d in subset)

    cand = [d for d in drills if d["tier"] <= max_tier]
    best = None

    for k in range(1, 4):  # 1..3 is enough for 3 segments
        for subset in combinations(cand, k):

            for d in subset:

            ok_targets = all(delta[s] <= -1 for s in P)
            ok_nontargets = all(delta[s] <= 0 for s in set(SEGMENTS) - P)

            if ok_targets and ok_nontargets:
                if best is None or objective(subset) < objective(best):
                    best = subset
        if best is not None:
            break

    return best  # minimal-size subset, tie-broken by objective
```

### 3) Make it _truly minimal in practice_: retest‑driven stopping

The theoretical minimal set is only “expected minimal.” To get **actual minimal**, you do this:

1. Select minimal set $X$.
2. Execute one drill.
3. Retest the segment that drill is supposed to flip.
4. If it flips, remove that segment from $P$.
5. Recompute minimal set for remaining $P$.
6. Stop as soon as $P=\emptyset$.

This converts a static set cover into an **adaptive minimal sequence**.

**Pseudocode (adaptive):**

```python
def order_drills(subset):
    # recommended execution order: calm cervical -> organize thorax -> organize lumbar -> integrate locomotion
    priority = {"C": 1, "T": 2, "L": 3, "global": 4}
    return sorted(subset, key=lambda d: d["priority"])

def adaptive_min_protocol(P, drills, max_tier=1, test_fn=None, execute_fn=None):
    # test_fn(segment)-> True if segment is now OK (negative gradient proxy)
    # execute_fn(drill)-> performs dose (your implementation)

    protocol = []

    while P:
        subset = choose_min_set(P, drills, max_tier=max_tier)
        if subset is None:
            # relax constraints: allow non-target positives with penalty, or increase tier
            subset = choose_min_set(P, drills, max_tier=max_tier+1)
            if subset is None:
                return protocol, P  # couldn't solve with library/tier

        subset = order_drills(subset)

        # Execute drills one-by-one, retesting after each
        for d in subset:
            protocol.append(d["name"])
            execute_fn(d)

            # Retest only segments the drill is intended to improve (those with v=-1)
            improved_segments = [s for s in P if d["v"][s] < 0]
            for s in improved_segments:
                if test_fn(s):  # segment flipped
                    P.remove(s)

            if not P:
                break

    return protocol, set()
```

## D) Precomputed minimal sets for each inversion pattern (Tier‑0/1 safe defaults)

Because there are only 3 segments, you can also hard‑code a minimal lookup table (this is what your enumeration will typically return anyway).

Let $P$ be the inverted segment set:

1. $P=\{L\}$ → **{ERBL}** (or {BSTEP})
2. $P=\{T\}$ → **{WSX}** (or {HANG‑S})
3. $P=\{C\}$ → **{SB}** (or {B90})
4. $P=\{L,T\}$ → **{BW}** (or {BSD})
5. $P=\{T,C\}$ → **{HANG‑S}** (or {WSX} or {B90})
6. $P=\{L,C\}$ → **{ERBL + SB}** (two drills; Tier‑0 safe)
7. $P=\{L,T,C\}$ → **{BW + HANG‑S}** (two drills; Tier‑0/1 safe)
    * Alternative two‑drill solution: **{BW + B90}**

Tier‑2 “single drill” solutions exist for patterns 6–7 (e.g., CRAWL or SCARRY), but they’re **not** reliably safe for A‑locked systems, so the algorithm should only pick them when `max_tier >= 2` _and_ your retest history shows they are stable.

## E) How to define the “test_fn(segment)” in your system

To make the algorithm executable, you need a segment→test mapping:

* **Lumbar test** $T_L$: backward step acceptance (quiet spine, no anterior hip pinch)
* **Thoracic test** $T_T$: arm raise without rib flare/neck recruitment
* **Cervical test** $T_C$: breath probe without SCM/shoulder rise (or neck quiet during arm raise)

The algorithm does not care which exact tests you use—only that each test is a consistent proxy for whether that segment is still acting as a load sink.

## F) Dosing rule (so selection is comparable)

Keep drill “dose” standardized so a “cost” of 1 means something:

* Breathing drill: **4–6 slow breaths**
* Hang: **15–30 seconds** supported
* ERBL: **3 slow reps per side**
* Backward walking: **45–90 seconds**
* Step-down: **5 reps per side**

Then retest immediately.

This is what makes the adaptive stopping meaningful.

## G) What makes this model self-improving

Once you start logging results, you can upgrade the signature from deterministic signs to probabilities:

Then selection becomes “minimal expected drills to reach target probability.” But the **sign‑matrix + retest loop** is the correct first version—and it already behaves like a control system.
